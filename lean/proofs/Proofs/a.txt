Happy to slow it down. The whole thing comes down to one bookkeeping question about the expansion of fⁿ: how many times did each position i get used? Let me build it up from there.
Expand the power by usage counts. Recall f(π) = x⁰y^{π(0)} + x¹y^{π(1)} + ⋯ + x^{N−1}y^{π(N−1)} has exactly one term per position i. Raising to the nth power means picking one term out of each of the n factors and multiplying them. The resulting monomial only depends on how many times each position was picked — say position i is picked cᵢ times, where c₀ + c₁ + ⋯ + c_{N−1} = n. A given count vector c = (c₀, …, c_{N−1}) arises in n!/(c₀!⋯c_{N−1}!) ordered ways, and the term it produces has x-exponent Σᵢ i·cᵢ and y-exponent Σᵢ cᵢ·π(i). So
f(π)n=∑c: ∑ci=nn!∏ici!  x∑ii ci  y∑ici π(i).f(\pi)^n = \sum_{\substack{c\,:\ \sum c_i = n}} \frac{n!}{\prod_i c_i!}\; x^{\sum_i i\,c_i}\; y^{\sum_i c_i\,\pi(i)}.f(π)n=c: ∑ci​=n​∑​∏i​ci​!n!​x∑iici​y∑i​ci​π(i).
Now sum over all permutations with their signs and pull the c-sum to the outside:
g(n)=∑c : ∑ci=nn!∏ici!  x∑ii ci⋅S(c),S(c):=∑π∈SNsign⁡(π) y∑ici π(i).g(n) = \sum_{c\,:\,\sum c_i = n} \frac{n!}{\prod_i c_i!}\; x^{\sum_i i\,c_i}\cdot S(c),\qquad S(c) := \sum_{\pi \in S_N} \operatorname{sign}(\pi)\, y^{\sum_i c_i\,\pi(i)}.g(n)=c:∑ci​=n∑​∏i​ci​!n!​x∑i​ici​⋅S(c),S(c):=π∈SN​∑​sign(π)y∑i​ci​π(i).
The x-monomial is fixed by c and just goes along for the ride; all the cancellation happens inside S(c).
The signs kill any count vector with a repeat. This is the crux. Suppose two positions are used equally often, cₐ = c_b for some a ≠ b. Pair each permutation π with the permutation π′ that swaps its two output values at a and b (so π′(a) = π(b), π′(b) = π(a), and π′ agrees with π everywhere else). This pairing is an involution, and swapping two values flips the sign: sign(π′) = −sign(π). What does it do to the exponent Σᵢ cᵢπ(i)? Only the a- and b-terms move, and the change is
(cₐπ(b) + c_bπ(a)) − (cₐπ(a) + c_bπ(b)) = (cₐ − c_b)(π(b) − π(a)).
When cₐ = c_b this is zero — so π and π′ produce the same y-monomial but carry opposite signs. They cancel. Every permutation is annihilated by its partner, so S(c) = 0. (If you like linear algebra: S(c) = det[(y^{cᵢ})ʲ], a Vandermonde determinant, and cₐ = c_b makes two rows equal, forcing the determinant to vanish — same fact, dressed differently.)
So the only surviving count vectors are those with all cᵢ distinct.
Distinct counts can't be small. A surviving c needs c₀, …, c_{N−1} to be N distinct nonnegative integers. The smallest their sum can possibly be is 0 + 1 + 2 + ⋯ + (N−1) = N(N−1)/2, achieved only by the set {0, 1, …, N−1} (any other choice of N distinct nonnegatives drops one of these and picks up something ≥ N, raising the sum). But that sum is forced to equal n. Hence when n < N(N1)/2 there is simply no surviving count vector at all — every S(c) is zero, so g(n) = 0.
That is the lower half: rank ≥ N(N−1)/2. As a sanity check at N = 3: a triple of nonnegative integers summing to 2 can't be all-distinct, since the smallest distinct triple 0+1+2 already sums to 3. So at n = 2 every count vector repeats, and g(2) = 0 — which is exactly what the brute force showed.
And it's nonzero right at the threshold. At n = N(N−1)/2 the surviving count vectors are precisely the rearrangements of (0, 1, …, N−1). Among these, the x-exponent Σᵢ i·cᵢ is — by the rearrangement inequality — maximized strictly and uniquely by c = (0, 1, 2, …, N−1) (pair the biggest count with the biggest coefficient). So the very top x-power in g is contributed by that one count vector alone, with nothing to cancel against, and its coefficient is a positive constant times the genuine Vandermonde ∏_{i<j}(yʲ − yⁱ) ≠ 0. Hence g(N(N−1)/2) ≠ 0.
Below the threshold g vanishes; at the threshold it doesn't. So rank(K_{N,N}) = N(N−1)/2.
The one sentence to remember: powering f records usage counts of positions, and the alternating signs wipe out every count pattern with a repeat — so g can't survive until the counts are forced to be N distinct numbers, which first becomes possible at 0 + 1 + ⋯ + (N−1).
